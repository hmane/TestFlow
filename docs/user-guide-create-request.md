# How to Create a New Request

This guide walks you through the process of submitting a new legal review request in the Legal Review System.

---

## Step 1: Navigate to Create a New Request

From the Home Dashboard, locate the toolbar at the top of the page.

**[SCREENSHOT: Home page toolbar showing the "New Request" link/button]**

Click on **"New Request"** to begin creating your request.

---

## Step 2: Select Request Type

You will see the Request Type selection screen with three options:

**[SCREENSHOT: Request Type Selector screen showing all three cards]**

### Available Request Types:

| Request Type | Description | Status |
|-------------|-------------|--------|
| **Communication** | External communications, marketing materials, presentations, and public-facing content requiring legal and compliance review. | Available |
| **General Review** | General legal review of documents, contracts, agreements, and other legal matters requiring attorney oversight. | Coming Soon |
| **IMA Review** | Investment Management Agreement review for institutional clients requiring specialized compliance review. | Coming Soon |

### What Communication Review Covers:
- Marketing materials
- Client presentations
- Website & social content
- Email campaigns

**[SCREENSHOT: Communication card selected with checkmark indicator]**

> **Note:** Currently, only **Communication Review** is available. General Review and IMA Review are planned for future releases.

### Workflow Preview

After selecting a request type, a workflow preview appears at the bottom showing the stages your request will go through:

**[SCREENSHOT: Workflow Preview section showing the stepper with all stages]**

Click **Continue** to proceed to the request form.

---

## Step 3: Complete the Request Form

The request form contains several sections organized into cards. Complete each section as required.

### 3.1 Request Information Card

This is the main card containing all required information about your request.

**[SCREENSHOT: Request Information card header with icon]**

#### Basic Information Section

**[SCREENSHOT: Basic Information section showing all fields]**

| Field | Required | Description |
|-------|----------|-------------|
| **Request Title** | Yes | A brief, descriptive title for your request |
| **Submission Type** | Yes | Select "New" or "Material Updates" |
| **Submission Item** | Yes | Select the type of material being submitted (e.g., New Exhibit, White Paper, Marketing Communication) |
| **Purpose** | Yes | Describe the purpose and context of this communication |
| **Target Return Date** | Yes | When you need the review completed |
| **Review Audience** | Yes | Select who needs to review: Legal, Compliance, or Both |

> **Rush Requests:** If your Target Return Date is earlier than the standard turnaround time for your Submission Item type, the request will automatically be flagged as a **RUSH** request. You will be prompted to provide a Rush Rationale.

**[SCREENSHOT: Rush Request indicator and Rush Rationale field when visible]**

#### FINRA Audience & Product Section

**[SCREENSHOT: FINRA Audience & Product section with all dropdowns]**

| Field | Description |
|-------|-------------|
| **FINRA Audience Category** | Select Institutional and/or Retail/Public |
| **Audience** | Select the target audience(s) for this communication |
| **U.S. Funds** | Select applicable U.S. fund(s) if any |
| **UCITS** | Select applicable UCITS fund(s) if any |
| **Separate Account Strategies** | Select applicable strategies |
| **Separate Account Strategies Includes** | Select what the strategy includes |

#### Distribution Method Section

**[SCREENSHOT: Distribution Method section with checkboxes]**

Select how this communication will be distributed:
- Dodge & Cox Website - U.S.
- Dodge & Cox Website - Non-U.S.
- Third Party Website
- Email / Mail
- Mobile App
- Display Card / Signage
- Hangout
- Live - Talking Points
- Social Media

#### Prior Submissions Section

**[SCREENSHOT: Prior Submissions section with picker and notes field]**

| Field | Description |
|-------|-------------|
| **Prior Submissions** | Link to any related previous requests that provide context |
| **Prior Submission Notes** | Add notes explaining the relationship to prior submissions |

#### Additional Parties Section

**[SCREENSHOT: Additional Parties section with people picker]**

| Field | Description |
|-------|-------------|
| **Additional Party** | Add any additional people who should have visibility into this request |
| **Date of First Use** | When the material will first be used (if known) |

---

### 3.2 Approvals Card

Before submitting for legal review, certain approvals may be required.

**[SCREENSHOT: Approvals card showing the approval grid]**

| Approval Type | When Required |
|--------------|---------------|
| **Communications Approval** | For marketing communications |
| **Portfolio Manager Approval** | For investment-related content |
| **Research Analyst Approval** | For research-related content |
| **SME Approval** | For subject matter expert review |
| **Performance Approval** | For content containing performance data |
| **Other Approval** | For any other required approvals |

For each required approval:
1. Check the checkbox to indicate approval is needed
2. Enter the approval date
3. Select the approver
4. Upload the approval document

**[SCREENSHOT: Single approval row expanded showing date, approver, and upload fields]**

---

### 3.3 Documents Card

Upload all documents related to your request.

**[SCREENSHOT: Documents card with upload area]**

**Supported Actions:**
- Drag and drop files into the upload area
- Click to browse and select files
- View, download, or delete uploaded documents

> **Important:** At least one document must be attached before submitting your request.

**[SCREENSHOT: Documents card with files uploaded showing the file list]**

---

## Step 4: Submit Your Request

Once all required fields are completed and documents uploaded:

### Actions Available

**[SCREENSHOT: Actions card/section showing Save Draft and Submit buttons]**

| Action | Description |
|--------|-------------|
| **Save as Draft** | Save your work and return later to complete |
| **Submit for Review** | Submit the request to begin the legal review workflow |

### Before Submitting

The system will validate your request. If any required fields are missing, you'll see error messages:

**[SCREENSHOT: Validation errors shown on the form]**

Fix any highlighted errors and try submitting again.

### After Submitting

Once submitted:
1. Your request status changes from **Draft** to **Legal Intake**
2. The Legal Admin team is notified
3. You can track progress from your Home Dashboard under "My Open Requests"

**[SCREENSHOT: Success message after submission]**

---

## Quick Reference: Required Fields

| Section | Required Fields |
|---------|----------------|
| **Basic Info** | Request Title, Submission Type, Submission Item, Purpose, Target Return Date, Review Audience |
| **Approvals** | At least one approval with date, approver, and document |
| **Documents** | At least one document uploaded |

---

## Tips for Faster Processing

1. **Be Descriptive:** Provide clear, detailed information in the Purpose field
2. **Plan Ahead:** Submit requests well before your Target Return Date when possible
3. **Complete Approvals:** Obtain all required approvals before submitting
4. **Link Prior Submissions:** If related to previous work, link those requests for context
5. **Upload All Materials:** Include all versions and supporting documents upfront

---

## Need Help?

If you have questions about creating a request, contact the Legal Admin team or refer to the system help documentation.

# Workflow Stages After Submission

Once you submit your request, it moves through several workflow stages. This section explains each stage, what you'll see, and what actions are available.

## Workflow Stepper Overview

The **Workflow Stepper** at the top of every request shows the current progress. Each step has a status indicator:

| Icon | Status | Description |
|------|--------|-------------|
| ○ | Not Started | Stage has not begun |
| ◐ | In Progress | Currently active stage |
| ✓ | Completed | Stage finished successfully |
| ⦻ | Not Required | Stage skipped (e.g., Compliance not needed) |

### Workflow Stages

**[SCREENSHOT: Workflow Stepper showing all stages]**

| Stage | Description |
|-------|-------------|
| **Draft** | Initial creation, not yet submitted |
| **Legal Intake** | Legal Admin reviews and assigns attorney |
| **Assign Attorney** | Committee assigns attorney (optional path) |
| **In Review** | Legal and/or Compliance reviews in progress |
| **Closeout** | Final steps before completion |
| **Completed** | Request fully processed |

> **Note:** Requests can also be put **On Hold** or **Cancelled** at various stages.

---

## Stage 1: Legal Intake

After submitting your request, the Legal Admin team receives it for intake processing.

**[SCREENSHOT: Legal Intake stage view]**

### What You See

From this point forward, the request displays:

1. **Request Summary Card** (Collapsible)
   - All submitted information in a compact, read-only format
   - Basic Info, Product & Audience, Distribution, Prior Submissions, Approvals
   - "Edit" button appears if you're the submitter (until Closeout)

2. **Documents Card**
   - All attached documents with preview/download options
   - Submitter can still add documents

3. **Comments Section** (Right side)
   - Threaded conversation about the request
   - All parties can add comments

### Legal Intake Form (Legal Admin Only)

Legal Admins see the **Legal Intake** card with these fields:

**[SCREENSHOT: Legal Intake form with attorney picker]**

| Field | Description | Who Can Edit |
|-------|-------------|--------------|
| **Review Audience** | Legal Only, Compliance Only, or Both | Legal Admin, Admin |
| **Assign Attorney** | Select from LW - Attorneys group | Legal Admin, Admin |
| **Assignment Notes** | Notes for the assigned attorney | Legal Admin, Admin |

### Available Actions

| Action | Who Can Perform | Description |
|--------|-----------------|-------------|
| **Assign Attorney** | Legal Admin, Admin | Directly assigns attorney and moves to In Review |
| **Submit to Assign Attorney** | Legal Admin, Admin | Sends to committee for attorney assignment |
| **Put On Hold** | Admin, Legal Admin, Owner | Pauses the request |
| **Cancel Request** | Admin, Legal Admin, Owner | Cancels the request |
| **Save** | Admin, Owner | Save changes (when form has edits) |
| **Close** | Everyone | Close the form |

---

## Stage 2: Assign Attorney (Optional)

If the Legal Admin sent the request to committee instead of directly assigning an attorney:

**[SCREENSHOT: Assign Attorney stage view]**

### What Attorney Assigners See

The **Legal Intake** card shows the same fields, but now the **Attorney Assigner** group can select an attorney.

### Available Actions

| Action | Who Can Perform | Description |
|--------|-----------------|-------------|
| **Assign Attorney** | Attorney Assigner, Admin | Assigns attorney and moves to In Review |
| **Put On Hold** | Admin, Legal Admin, Owner | Pauses the request |
| **Cancel Request** | Admin, Legal Admin, Owner | Cancels the request |
| **Close** | Everyone | Close the form |

---

## Stage 3: In Review

This is where the actual legal and/or compliance reviews happen, based on the **Review Audience** setting.

**[SCREENSHOT: In Review stage with Legal Review form]**

### What You See

1. **Request Summary Card** (Collapsible) - Read-only summary
2. **Legal Intake Card** (Completed) - Shows green "✓ Completed" header with assigned attorney

### Legal Review Form (When Review Audience = Legal or Both)

Attorneys see the **Legal Review** card with:

**[SCREENSHOT: Legal Review form with outcome selection]**

| Field | Description | Who Can Edit |
|-------|-------------|--------------|
| **Review Outcome** | Approved, Approved With Comments, Respond To Comments And Resubmit, Not Approved | Assigned Attorney |
| **Review Notes** | Detailed notes, comments, recommendations (append-only) | Assigned Attorney |

#### Legal Review Actions

| Action | Who Can Perform | Description |
|--------|-----------------|-------------|
| **Submit Review** | Assigned Attorney | Submits the legal review with selected outcome |
| **Save** | Assigned Attorney | Saves progress without submitting |

### Compliance Review Form (When Review Audience = Compliance or Both)

Compliance Users see the **Compliance Review** card with:

**[SCREENSHOT: Compliance Review form with all fields]**

| Field | Description | Who Can Edit |
|-------|-------------|--------------|
| **Foreside Review Required** | Toggle - Yes/No | Compliance Users |
| **Retail Use** | Toggle - Yes/No | Compliance Users |
| **Review Outcome** | Same options as Legal Review | Compliance Users |
| **Review Notes** | Detailed notes (append-only) | Compliance Users |

> **Important:** If either "Foreside Review Required" or "Retail Use" is set to Yes, a Tracking ID will be required during Closeout.

#### Compliance Review Actions

| Action | Who Can Perform | Description |
|--------|-----------------|-------------|
| **Submit Review** | Compliance Users | Submits the compliance review with selected outcome |
| **Save** | Compliance Users | Saves progress without submitting |

### Review Status Sub-States

During review, the status can show:

- **In Progress** - Reviewer is working on it
- **Waiting On Attorney** - Legal review awaiting attorney action
- **Waiting On Compliance** - Compliance review awaiting team action
- **Waiting On Submitter** - Reviewer requested additional info from submitter

### General Actions During In Review

| Action | Who Can Perform | Description |
|--------|-----------------|-------------|
| **Put On Hold** | Admin, Legal Admin, Owner | Pauses all reviews |
| **Cancel Request** | Admin, Legal Admin, Owner | Cancels the request |
| **Save** | Admin, Owner | Save any changes to request info |
| **Close** | Everyone | Close the form |

### Review Outcomes and Next Steps

| Outcome | What Happens |
|---------|--------------|
| **Approved** | Proceeds to next review (if Both) or Closeout |
| **Approved With Comments** | Same as Approved, comments captured |
| **Respond To Comments And Resubmit** | Returns to submitter for changes |
| **Not Approved** | Moves directly to Completed (bypasses Closeout) |

---

## Stage 4: Closeout

When all required reviews are completed with Approved or Approved With Comments, the request moves to Closeout.

**[SCREENSHOT: Closeout stage view]**

### What You See

1. **Request Summary Card** (Collapsible, Read-only)
2. **Legal Intake Card** (Completed) - Green header
3. **Legal Review Card** (Completed) - Green header showing outcome, duration, notes
4. **Compliance Review Card** (Completed, if applicable) - Green header
5. **Closeout Card** (In Progress)

### Closeout Form

**[SCREENSHOT: Closeout form with Tracking ID field]**

| Field | Description | Required When |
|-------|-------------|---------------|
| **Tracking ID** | Unique identifier for the approved communication | Required if Foreside Review or Retail Use = Yes |
| **Closeout Notes** | Final notes or comments (append-only) | Optional |

### Available Actions

| Action | Who Can Perform | Description |
|--------|-----------------|-------------|
| **Complete Closeout** | Admin, Owner (Submitter) | Finalizes the request, moves to Completed |
| **Close** | Everyone | Close the form |

> **Note:** On Hold and Cancel are not available during Closeout.

---

## Stage 5: Completed

The request has been fully processed and is now in its final state.

**[SCREENSHOT: Completed request view]**

### What You See

All cards are in read-only mode with green "✓ Completed" headers:

1. **Request Summary Card** - Final submitted information
2. **Legal Intake Card** - Shows attorney assignment details
3. **Legal Review Card** - Shows outcome, reviewer, duration, notes
4. **Compliance Review Card** (if applicable) - Shows outcome, flags, notes
5. **Closeout Card** - Shows Tracking ID, completion date

### Available Actions

| Action | Who Can Perform | Description |
|--------|-----------------|-------------|
| **Close** | Everyone | Close the form |

No edits are possible after completion.

---

## Special States

### On Hold

A request can be put on hold at most workflow stages. When on hold:

**[SCREENSHOT: On Hold request view with Resume button]**

- The Workflow Stepper shows a pause indicator
- All editing is suspended
- Time tracking is paused
- A "Hold Reason" is captured and displayed

#### Resuming from Hold

| Action | Who Can Perform | Description |
|--------|-----------------|-------------|
| **Resume** | Admin, Legal Admin, Owner | Returns request to previous status |

### Cancelled

A cancelled request:

**[SCREENSHOT: Cancelled request view]**

- Shows "Cancelled" status in stepper
- Displays cancellation reason
- All cards are read-only
- Cannot be resumed

## Time Tracking

The system automatically tracks time spent at each workflow stage:

| Metric | Description |
|--------|-------------|
| **Legal Intake Hours** | Time from submission to attorney assignment |
| **Legal Review Hours** | Attorney time + Submitter response time |
| **Compliance Review Hours** | Reviewer time + Submitter response time |
| **Closeout Hours** | Time to complete closeout |
| **Total Reviewer Hours** | Sum of all reviewer/admin time |
| **Total Submitter Hours** | Sum of all submitter response time |

Time is calculated in **business hours** (8 AM - 5 PM PST, Monday-Friday).
